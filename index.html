Interview questions bank

Q1 — Most asked everywhere

What is the difference between WHERE and HAVING?

WHERE filters rows before GROUP BY. HAVING filters after aggregation. WHERE cannot use aggregate functions; HAVING can.
Q2 — Basic joins

What is the difference between INNER JOIN and LEFT JOIN?

INNER JOIN returns only matching rows from both tables. LEFT JOIN returns ALL rows from the left table + matching from right (unmatched = NULL).
Q3 — Window functions

What is the difference between ROW_NUMBER, RANK, and DENSE_RANK?

ROW_NUMBER — unique sequential numbers, no ties. RANK — ties get same number, next rank skipped. DENSE_RANK — ties get same number, no gaps. Use DENSE_RANK for top-N per group queries.
Q4 — NULL handling

How do you handle NULL values in SQL?

Use IS NULL or IS NOT NULL to check (never = NULL). Use ISNULL(col, 0) or COALESCE(col, 'default') to replace. NULLs are excluded from aggregate functions.
Q5 — Query execution order

What is the execution order of SQL clauses?

1. FROM (+ JOINs) → 2. WHERE → 3. GROUP BY → 4. HAVING → 5. SELECT → 6. ORDER BY → 7. LIMIT/TOP
This is why you cannot use a SELECT alias in a WHERE clause — WHERE runs before SELECT.
Q1 — VLOOKUP troubleshooting

Your VLOOKUP is returning #N/A for some values even though data looks correct. What are the possible reasons?

1. Leading/trailing spaces — use TRIM(). 2. Data type mismatch — number stored as text. 3. 4th argument TRUE instead of FALSE. 4. Lookup value not in first column — switch to INDEX-MATCH.
Q2 — INDEX-MATCH vs VLOOKUP

When would you prefer INDEX-MATCH over VLOOKUP?

INDEX-MATCH is more flexible: lookup column need not be the first, can look left, handles column insertions without breaking, and is faster on large datasets.
Q3 — Data cleaning workflow

Walk me through your full data cleaning process before building a report.

1. Freeze header row. 2. Remove duplicates on key column. 3. Check data types (dates, numbers). 4. TRIM() text fields. 5. Handle blanks with IFERROR/IF(ISBLANK). 6. Validate ranges. 7. Keep raw data untouched — work on a copy.
Q1 — DAX context

What is the difference between filter context and row context in DAX?

Row context: exists inside calculated columns/iterating functions — evaluates each row individually. Filter context: exists inside measures — the set of filters from slicers/visuals currently applied. CALCULATE() modifies filter context.
Q2 — Measures vs calculated columns

When do you use a DAX measure vs a calculated column?

Measures: computed at query time based on filter context — use for aggregations (SUM, AVERAGE, %). Calculated columns: stored in model row by row — use for row-level categorization. Prefer measures for performance.
Q3 — % of total DAX

Write a DAX measure to calculate each product's % contribution to total sales.

% of Total =
DIVIDE(
  SUM(Sales[Amount]),
  CALCULATE(SUM(Sales[Amount]), ALL(Products[ProductName]))
)
Q1 — Mean vs Median

Why is median a better measure than mean for salary data?

Mean is distorted by outliers (very high salaries). Median gives the true middle value — so if 90% earn ₹4–6 LPA but a few earn ₹50+ LPA, median ₹5 LPA is far more representative than an inflated mean.
Q2 — Correlation vs causation

Ice cream sales and drowning rates are correlated. Does ice cream cause drowning?

No — both are caused by hot weather (a confounding variable). People buy ice cream AND swim more in summer. Always look for the confounding variable before concluding causation.
Q3 — Outlier detection

How do you detect and handle outliers?

IQR method: Q3 − Q1 = IQR. Outlier if value < Q1 − 1.5×IQR or > Q3 + 1.5×IQR. Handle by: investigate first (data error vs genuine extreme), then cap, remove, or flag depending on context.

MediumWhat is the difference between ROW_NUMBER(), RANK(), and DENSE_RANK()?EXL classic▾
Model answer

ROW_NUMBER(): Always unique — 1,2,3,4 even if tied. No concept of ties. Good for deduplication (pick one row per group).

RANK(): Ties get same rank, next rank is skipped. Ties at rank 2: both get 2, next rank is 4. Olympic-style leaderboard.

DENSE_RANK(): Ties get same rank, no gaps. Ties at rank 2: both get 2, next rank is 3. Use for salary bands, product ratings.

Follow-up: "Which one for top 3 salaries per department?" → DENSE_RANK — because tied people at rank 2 should both appear in top 3.

Real business case studies

5 real-world analytics scenarios — practice your approach before interviews.

Case 1Sales decline analysis

Problem statement

An e-commerce company's total sales dropped 18% in Q3 2024 compared to Q3 2023. Management wants to understand the root cause and recover by Q4.

1
Validate the data: Confirm numbers across regions, channels, and time periods. Check for reporting errors first.
2
Break it down: Segment decline by region, product category, customer type, and sales channel.
3
YoY comparison: Compare Q3 2024 vs Q3 2023 for each segment to find where the drop is concentrated.
4
Check external factors: Competitor launch? Price increase? Seasonal shift?
5
Identify top contributors: 80/20 rule — usually 2–3 factors explain 80% of the decline.

Case 2Data discrepancy investigation

Problem statement

Two reports from different teams show different revenue figures for the same month. Which one is correct?

1
Identify the data source for each report — same table or different tables?
2
Check the date filter logic — is one using order date, the other using payment date?
3
Check for cancelled/returned orders — one report may include them, the other may not.
4
Reconcile row by row with a VLOOKUP on transaction IDs to find the exact difference.
Always document your findings and create a single source of truth. Communication and transparency are more important than being "right."

Case 3MIS dashboard design

Problem statement

Design a monthly MIS dashboard for a 500-person sales team. What metrics would you track?

1
KPIs to track: Total revenue vs target (variance %), top/bottom 10 reps, conversion rate, average deal size, MoM and YoY growth.
2
File structure: Tab 1 — Raw Data (locked), Tab 2 — Cleaned Data, Tab 3 — Pivot Tables, Tab 4 — Dashboard, Tab 5 — Trend (6-month).
3
Design principles: Dashboard shows numbers only — no raw data. Green = above target, Red = below. Protect sheets.

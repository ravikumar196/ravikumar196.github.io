Q1

Difference between WHERE and HAVING?

WHERE filters rows before GROUP BY. HAVING filters after aggregation.
Q2

INNER JOIN vs LEFT JOIN?

INNER JOIN returns only matching rows. LEFT JOIN returns all left-table rows.
Q3

ROW_NUMBER vs RANK vs DENSE_RANK?

ROW_NUMBER — unique always. RANK — ties skip next rank. DENSE_RANK — ties, no gaps.
Q4

SQL execution order?

FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT
Q1

VLOOKUP returning #N/A — reasons?

1. Leading/trailing spaces — use TRIM(). 2. Data type mismatch. 3. 4th argument TRUE. 4. Lookup not in first column.
Q2

INDEX-MATCH vs VLOOKUP?

INDEX-MATCH can look left, doesn't break on column insertions, faster on large datasets.
Q1

Filter context vs row context in DAX?

Row context: inside calculated columns/iterators. Filter context: inside measures — active slicer/visual filters.
Q2

Measure vs calculated column?

Measures: computed at query time. Calculated columns: stored row by row. Prefer measures for performance.
Q1

Correlation vs causation — classic example?

Ice cream sales and drowning rates correlate — but hot weather causes both.
Q2

How to detect outliers?

IQR method: outlier if value < Q1 − 1.5×IQR or > Q3 + 1.5×IQR.
